# 添加订单详情API端点 - 交互式计划制定示例

## 场景描述

这是一个电商系统的后端开发任务。系统当前已有订单列表API，但缺少获取单个订单详情的端点。需要添加一个新的API端点来支持前端展示订单详情页面。

**技术背景**：
- 后端框架：Node.js + Express
- 数据库：MongoDB + Mongoose
- 认证方式：JWT Token
- 现有代码库：已有用户认证、订单列表等基础功能

## 用户初始请求

> 用户：我需要添加一个获取订单详情的API端点，前端需要根据订单ID查询订单的完整信息。

## 第2轮：方案设计与细化

### AI 分析

基于需求，设计以下技术方案：

**路由设计**
- 路径：`GET /api/orders/:id`
- 中间件链：`authMiddleware` → `getOrderDetail` → 错误处理

**数据库查询**
- 使用 Mongoose 的 `populate` 方法关联查询商品详情
- 查询订单时同时获取关联的商品信息

**权限验证**
- 在 authMiddleware 中验证JWT Token，获取当前用户ID
- 在 getOrderDetail 中检查订单的userId是否匹配

**错误处理**
- 使用统一的错误处理中间件
- 返回标准化的错误响应格式
