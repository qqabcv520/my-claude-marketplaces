# Project Spec

项目规范管理插件，提供项目文档初始化和交互式计划制定功能。

## 功能特性

### 1. 项目文档初始化 (`/ps-init`)

快速初始化项目的文档结构，自动生成 CLAUDE.md 和 docs/ 目录下的各类文档。

**主要功能：**
- 智能项目识别（前端/后端/Python/数据分析/Claude插件等）
- 动态文档生成（根据项目类型推荐合适的文档清单）
- 文档分层设计（CLAUDE.md 只包含高频使用内容，详细文档放到 docs/）
- 增量更新支持（检测已存在文件，避免覆盖用户内容）

### 2. 交互式计划制定 (`/ps-plan`)

通过多轮对话创建详细、可执行的实施计划。

**主要功能：**
- 5 阶段规划流程（深度理解 → 方案设计 → 审查对齐 → 最终计划 → 退出规划）
- 按复杂度裁剪需求澄清、功能拆解和技术规约
- 输出可供 `/ps-exec` 消费的结构化计划文件
- 技术方案评估和权衡
- 风险评估和测试策略

### 3. 计划执行 (`/ps-exec`)

执行 `/ps-plan` 创建的计划，支持基于步骤依赖的批次执行、进度追踪和断点续执。

**主要功能：**
- 基于步骤依赖的批次执行（优先执行无依赖步骤）
- subAgent 协调（按步骤批次分发给独立 subAgent 执行）
- 并行执行（同一批次的无依赖步骤可并行运行）
- 进度追踪（执行记录写入独立的 `.task.md` 文件，支持实时更新和续执）
- 多层验证（实时验证 + 阶段验证 + 最终验证）
- 自动修复（验证失败时尝试自动修复）
- 断点续执（支持从中断处继续执行）

## 使用方法

### 初始化项目文档

```bash
/ps-init
```

### 创建实施计划

```bash
/ps-plan [可选：功能描述]
```

**示例：**
```bash
/ps-plan 添加用户认证功能
/ps-plan 实现深色模式
/ps-plan
```

### 执行计划

```bash
/ps-exec [可选：计划文件路径]
```

**示例：**
```bash
/ps-exec docs/spec/20260311-1030-user-auth.md
/ps-exec  # 自动选择最新的计划文件
```

## 生成的文档结构

```
项目根目录/
├── CLAUDE.md              # 精简版项目文档
└── docs/
    ├── spec/              # 实施计划（由 /ps-plan 生成）
    ├── ARCHITECTURE.md    # 架构设计（按需）
    ├── TESTING.md         # 测试规范（按需）
    ├── API.md             # API 文档（按需）
    └── ...                # 其他文档（按需）
```
